# Content review — curriculum-designer
Reviewer: curriculum-designer — evidence-based instructional design for adult professionals
Date: 2026-06-09
Scope read: all 16 rendered units + requirements §1 + workflows.md + learner docs

## Headline
The unit skeleton is genuinely strong: objectives are behavioral, every lab maps to its objectives, self-checks are objective, scaffolding fades sensibly across the arc (U1 hands Sam the literal prompt; U16 hands a goal), and the misconception-preemption in "Common pitfalls" is some of the best LLM-mental-model repair work I've seen in self-paced material. But the course is missing the two things the learning-science literature says a self-paced solo course cannot do without: **real worked examples** (the learner never sees a single transcript of an actual Claude session — every "Worked example" is a description of the course's own build, not a demonstration at the learner's level) and **any consolidation mechanism between U1 and the capstone** (no retrieval practice, no spaced review, no formative checkpoint — later units assume habits "built" earlier that nothing ever verified took hold). On top of that, the single-sourced version-data tokens are pasted into sentences as reference-card fragments, producing broken grammar and verbatim triple-repetition inside single units — pure extraneous cognitive load that also undermines the prose's credibility. Net: the architecture is excellent, the practice design is good, the *exposition layer* is where the "missing something" lives.

## Cross-cutting findings

**F1 — The learner never sees Claude Code actually being used; "Worked example" sections describe artifacts instead of demonstrating work. Severity: high.**
Evidence: U1's worked example is the `doctor` script's four checks (01-onboarding-first-win/unit.md:87–109); U5's is "This repository is itself a worked example of the loop" (05-ship-a-feature/unit.md:99); U6's is the course's own check suite (06-tdd/unit.md:103–114); U7's is a `check-version-refs` build anecdote (07-debug-a-failure/unit.md:100–110); U10, U11, U12, U14, U16 likewise point at the course's own spec/hooks/CI. Nowhere in 16 units is there a worked trace of the actual skill being taught: prompt → Claude's response → the diff → the verification step. For an audience that is *expert in engineering but novice in agentic coding* (requirements.md:31), the worked-example effect predicts exactly this gap hurts most: the novice-side skill (what a good session looks like, turn by turn) is the one never modeled. The dogfooding examples are also self-referential extraneous load — U7's worked example requires understanding `vd:` tokens, `check-version-refs`, and the course's authoring pipeline, none of which Sam has any reason to know (07-debug-a-failure/unit.md:100–110). Exceptions that show the right shape: U2's sample architecture summary (02-explore-a-codebase/unit.md:79–90) and U13's concrete agent JSON (13-subagents/unit.md:130–139). Principle: worked-example effect; example–problem pairing before independent practice.

**F2 — Version-data token substitution grafts reference-card fragments into instructional sentences, breaking grammar and repeating verbatim text up to three times per unit. Severity: high.**
Evidence: U4: "Project memory via CLAUDE.md, auto-discovered at startup (auto-memory). A `CLAUDE.md` is a file…" (04-memory-and-context/unit.md:56) — a sentence fragment dropped mid-paragraph. U4:85: "Inspect and reset it. In-REPL slash commands: /context, /compact, /clear (inspect and manage the context window). You can see what's currently loaded…". U3:60: "The currently available modes are acceptEdits, auto, bypassPermissions, default, dontAsk, plan (`--permission-mode <mode>`) — the exact set is version-specific". U12 pastes the identical ~70-word commands blob (including the "$1 is the SECOND arg" gotcha) **three times** in one unit (12-commands-and-skills/unit.md:65, 97, 217); U11 pastes the `ultrareview` blob three times (11-code-and-security-review/unit.md:70, 111, 207); U16's Going-deeper bullet is three blobs concatenated (16-automate-and-scale/unit.md:224). Single-sourcing the *facts* is right; rendering the same reference string into running prose at every mention is the redundancy effect plus coherence violation — every repetition is reading time spent on something already read, and the seams ("(auto-memory)") read as machine output, eroding trust in AI-authored prose for a skeptical audience. Principle: extraneous cognitive load (redundancy/coherence, Mayer); signaling.

**F3 — No retrieval or consolidation mechanism exists between unit N and the capstone; later units assume consolidation nothing ensured. Severity: high.**
Evidence: U5 opens "You can now land a change, read unfamiliar code, operate safely, and steer Claude with memory and context" (05-ship-a-feature/unit.md:50–51) — asserted, never checked. The Skip-checks are framed only as *skip* decisions ("Skip this unit if you can already…", every unit), not as retrieval prompts on entry; the progress checklist is an honor-system tick list (progress-checklist.md:7–9); the sole assessment is the capstone (requirements.md:21, "per-unit graded assessments" explicitly out of scope at :28). The CV mantra is well spaced (it genuinely recurs in every unit — good), but the *procedural* skills (plan-mode gating, characterization tests, hook payload testing) are each practiced exactly once and never revisited. For a solo learner the literature predicts illusion of mastery: a green `verify-lab` once is recognition-level evidence, and nothing prompts re-retrieval days later. Even 3–5 retrieval questions at unit close ("Without looking: what are the two gates in W1? Why must a red test fail for the *right* reason?") would close most of this at near-zero cost. Principle: retrieval practice / spacing; testing effect.

**F4 — Transfer to the learner's own work is never designed; every rep happens on taskflow or the course repo. Severity: high.**
Evidence: the only bridges to Sam's real environment are an optional stretch in U8 ("on your own fork/remote…", 08-git-and-pr/unit.md:147), a parenthetical in U12 ("or two of your own you genuinely repeat", 12-commands-and-skills/unit.md:162), and U15's "either a real one you'd actually use" (15-mcp-and-vetting/unit.md:173). No unit ends with a "now on your repo" transfer task: write a CLAUDE.md for *your* project (U4 is the obvious candidate and doesn't ask), run the light-onboarding loop on a repo you actually maintain (U2), wire a hook for *your* team's real standard (U14). For time-pressed senior engineers who "resent hand-holding" and "value their time" (requirements.md:31), own-work application is both the transfer mechanism and the motivation hook, and the course leaves it entirely to the capstone — one event, at the end. Principle: near→far transfer design; adult-learning relevance (immediate applicability).

**F5 — AI-generation artifacts shipped in learner-facing files: stray tool-call XML at the end of four units. Severity: medium (trivial fix, outsized credibility cost).**
Evidence: `</content>` and `</invoke>` literally present in the rendered files at 01-onboarding-first-win/unit.md:173–174, 02-explore-a-codebase/unit.md:157, 03-operate-safely/unit.md:215–216, 05-ship-a-feature/unit.md:190–191 (verified by byte inspection). A course that teaches "read the diff before accepting" shipping unread AI tool-wrapper residue in its own first unit is a direct hit to the persona it must convince ("skeptical of AI output", requirements.md:31). Principle: modeling/credibility; the medium contradicting the message.

**F6 — Prose density is high and stylistically monotone; reading-time labels understate it. Severity: medium.**
Evidence: nearly every Concept section is the same template — numbered "1 — … 2 — …" theses, each carrying a bolded multi-clause aphorism with em-dash chains ("the most costly mistake isn't a bug, it's *building the wrong thing correctly*", 10-spec-driven-dev/unit.md:59; this register is constant across all 16 units). Uniform emphasis flattens salience — when every other sentence is bold-italic load-bearing, none is. U13 is 275 dense lines labeled 13 min (13-subagents front matter:12); U12's 10 min includes the triple-pasted token blobs. Boilerplate repeated per unit (Version-currency paragraph, "Stuck?" footer) adds further fixed overhead. Intrinsic load here is legitimately high (new mental models); the extraneous share — rhetorical packing, redundancy — is what's controllable and isn't controlled. Principle: cognitive-load management; signaling/salience.

**F7 — Zero visual representations across 16 units; the course's central mental models are taught in prose only. Severity: medium.**
Evidence: the agentic loop — *the* foundational model — is one paragraph of prose (01-onboarding-first-win/unit.md:49–58); the context window (U4), the three permission postures (U3 — though its decision table at :131–136 is the one good quasi-visual), the subagent context-isolation picture (U13), and the defense-in-depth layers (U14:140–144) are all prime dual-coding candidates and all prose. CommonMark permits ASCII/Mermaid diagrams; the R15 no-meaning-by-color rule does not forbid structure. Principle: dual coding / multimedia principle.

**F8 — Several stated learning objectives have no practice or assessment behind them (objective–practice misalignment at the edges). Severity: low.**
Evidence: U5 LO4 "Recognize when to reach for extended thinking" (05-ship-a-feature/unit.md:30) — taught in one awareness callout (:88–91), lab explicitly doesn't exercise it (:91), nothing assesses it; yet it sits at equal rank with the unit's core objectives. U4 LO3 "Locate where settings live" (04-memory-and-context/unit.md:28) is exercised only incidentally. Awareness-tier content is fine (the depth-tier concept exists, requirements.md:52) — but listing it as a same-rank "by the end you can" claim miscalibrates the learner's self-assessment. Principle: constructive alignment (Biggs).

**F9 — A few predictable wrong mental models about LLM behavior go unpreempted. Severity: low-medium.**
Evidence: U4 advises `/clear` when "Claude seems confused" (04-memory-and-context/unit.md:86–88) but never explains *why* long sessions degrade (compaction, attention dilution) — a senior engineer will want the mechanism, and without it the advice is folklore. Cost/token economics are never taught anywhere before `--max-budget-usd` appears in U16 — a daily-driver concern for a professional. Non-determinism is explained once (U1:57–58) and never revisited when it matters more (why two runs of the same lab differ; why a flaky review finding isn't a contradiction). What *is* preempted is excellent — fluency≠correctness (U2:61–67), "re-asking the AI is not confirmation" (11-code-and-security-review/unit.md:196–197), "Claude resisted the injection is not the safeguard" (03-operate-safely/unit.md:193–195). Principle: misconception-driven design; the self-paced-solo requirement that text must repair what no instructor will.

**F10 — Judgment-graded labs (U8, U10, U12–U15) rely on self-applied checklists with no contrasting cases to calibrate against. Severity: low.**
Evidence: U10's rubric asks Sam to judge whether his own requirements are "testable… not implementation" (10-spec-driven-dev/unit.md:171–181) with only the full-scale course spec as a model — no small bad-spec/good-spec pair to calibrate the judgment the rubric demands. Same shape in U8 (:149–159) and U12 (:170–182). Solo learners are systematically miscalibrated self-assessors; contrasting cases (one flawed example annotated against the rubric) are the cheap fix. Principle: self-assessment calibration; contrasting-cases research.

## Top 5 enhancement recommendations

1. **Add one annotated session transcript per stage (4 total) as the Worked example.** A condensed, real prompt→response→diff→verify trace with margin notes ("here's where I rejected the plan and why"), starting with U5's explore→plan→code→commit. This converts F1 directly; U2:79–90 already shows the house style for a checkable model answer.
2. **Fix the token-rendering layer: render `{{vd:*}}` once per unit as a "Version surface" callout box, and have prose refer to it** ("the current mode names are in the box below") instead of inlining the reference string at every mention. Eliminates the broken-grammar seams and the verbatim triple-repetition (U12:65/97/217, U11:70/111/207) in one mechanical change to `tools/render-units`.
3. **Append a 3–5 item "Check yourself" retrieval block to every unit, plus a cumulative one at each stage boundary** (closed-book questions: "What are W1's two gates? Why must red be for the right reason?"). Cheap, fits the existing template, and is the minimum viable consolidation mechanism for a course whose only assessment is terminal (F3).
4. **Add a one-paragraph "On your codebase" transfer prompt at the end of each unit's lab** — U4: draft a 10-line CLAUDE.md for a repo you own and verify it loads; U2: run the three located-change-site questions on your own project; U14: wire your team's one real "must always" as a hook. Ungraded, but it converts the course from simulator-only to transfer-designed (F4).
5. **Strip the generation residue and internal-QA leakage from rendered units** — the `</content>`/`</invoke>` tails (U1:173, U2:157, U3:215, U5:190), the "(unverified — …)" aside (U3:82), and the stray blank-bullet lines (U12:28, U14:26) — and add a rendered-output lint to `make check` so artifacts of this class can't ship again. Smallest fix, largest credibility return for a skeptical-of-AI audience.

## Out-of-scope flags

- **Cost/token/model-selection literacy is a missing *outcome*, not missing teaching:** no can-do in the capability map covers estimating spend, choosing models, or rate-limit behavior; `--max-budget-usd` appears only in U16. For professionals this is a daily concern the course never promises. (Spec-level gap, not unit prose.)
- **Prompting skill itself is never a named capability.** Good prompts are modeled inside lab steps (quoted prompts in U1–U5), but "write an effective instruction/decompose a task for an agent" is never promised, taught, or assessed as its own outcome.
- **Session lifecycle management** (`--resume`/`--continue`, recovering interrupted work) — a real daily-driver surface with no promised outcome anywhere.
- **Single-terminal-assessment structure (capstone as sole assessment, requirements.md:21/:28)** is a spec-level decision that amplifies F3; any formative-assessment fix beyond unit prose (graded mid-course checkpoint) would require a requirements change. Flagged, not solved.
- **Labs and capstone themselves were not reviewed** per scope; F10's contrasting-cases gap touches lab assets and may be partly addressable there rather than in unit prose.
