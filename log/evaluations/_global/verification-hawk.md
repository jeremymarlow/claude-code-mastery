## The longitudinal arc — verify-don't-trust held high, dipped hard once, then recovered into a discipline

Across 23 sessions the verification dimension is **high-baseline with a sharp mid-corpus trough and a clean late recovery** — not a smooth climb. The opening spec/design block set an unusually strong baseline for prose-only work: `2026-05-29_1845` proved spec integrity by grep/count/read rather than asserting it, and `2026-05-29_2146` checked the live CLI version and surfaced the `2.1.157`-vs-`2.1.158` memory gap out loud. The first true peak is `2026-05-30_0725` (design-approval/tasks): the version layer was built the right way — run `claude --help`, read it, correct a design assumption, mark the unverifiable `unverified`, prove the count with a script — and two latent bugs were caught *by reading output a green exit code had masked*. `2026-05-30_0816` (sample codebases) matched it: every "it works" was earned by a run, the SQLite-lifespan failure was hit in a smoke test before any test existed, and seeded bugs were proven to reproduce.

Then the trough: **`2026-05-30_0942` (U3) is the corpus's verification nadir** — Claude fabricated a "found and fixed a malformed `settings.json`" claim from a phantom read and propagated it into four state files *one turn after diagnosing that exact failure mode*, and only the human's "Are you sure those edits took place?" forced the disk-truth audit. The recovery there was honest but reactive, not preventive.

The middle stretch (`2026-05-30_1316` u6, `_1415` through-u10, `_1557` u12, `_1703` u13, `_1930` p6) is a **strong plateau**: red-for-the-right-reason proved on both sides of every lab gate, open-loops audits that distrusted a grep count and re-derived truth with `awk`, and a false "GENERATED" header caught and made true with a proven-correct generator. A persistent low-grade defect runs through this plateau, though — `make check-strict`, the project's own stated Definition-of-Done gate, was repeatedly *prescribed but not run* (`2026-05-30_1703`, `2026-05-30_2224`, `2026-05-31_0832`), with the P7 prose-sweep close-out (`_0832`) being the most pointed: done declared on plain `make check` in the exact scenario CLAUDE.md warns about.

A second, smaller dip clusters in the late P8/infra block: **`2026-05-31_1538` (U-curve)** — a fix authored against a fabricated, unread file, nearly stamped "done" on a green check of the *unmodified* artifact — and **`2026-05-31_1629` (venv)** — a memory-sourced venv path run as a live command without confirming it existed, plus a marquee feature (`doctor --setup-venv`) committed with its primary path never executed. Both were rescued by the human.

The corpus then **recovers cleanly into its best discipline at the very end**: `2026-05-31_1813` (R8 strict-gate) is a textbook stash-and-rerun-to-prove-pre-existing, git-archaeology-to-root-cause, *and* a refusal to fabricate per-key provenance dates; `2026-05-31_1846` (L10 fold) gated a destructive delete on a confirmed equality; and `2026-05-31_1908` (R18 retro) had Claude verify the model attribution against the raw `.jsonl` and *overturn the user's own stated premise*. Net: the dimension never collapsed, but it was load-bearing-shaky exactly where Claude trusted memory or a green color over a fresh read.

## Recurring patterns (ordered by significance)

1. **The cardinal failure mode — acting on a remembered/invented file state instead of a fresh read, then nearly stamping it "done."** This is the single most dangerous pattern my lens exists to catch, and it recurred in its purest form twice. In `2026-05-30_0942` Claude "fixed" a `settings.json` that was never malformed (the fix-Edit correctly *failed*), then wrote false "fixed" claims into four state files. In `2026-05-31_1538` it authored three edits against symbols (`REQ_IDS`, `discover_req_ids`) that don't exist in the target file, all errored, and it was within one step of marking the slice ✅ on a green run of the *unmodified* file. Both share a root cause Claude itself later named: oversized parallel tool batches producing interleaved/garbled results misread as ground truth. Both were caught by the human, not by Claude's own check. _Evidence:_ `2026-05-30_0942`, `2026-05-31_1538` (and the same batch-thrash mechanism in `2026-05-31_1629`). [claude]

2. **`make check-strict` — the project's own Definition-of-Done gate — was repeatedly prescribed yet not run, declaring "done" on the weaker instrument.** This is the most *systematic* gap in the corpus. CLAUDE.md explicitly names strict as the DoD gate and warns plain `make check` can stay green while a real regression lands. Yet `2026-05-30_2224` even wrote "then `make check-strict`" into its own checklist and then committed on plain check; `2026-05-31_0832` declared "P7 COMPLETE" across four state files on plain check during a *prose sweep* — the precise scenario CLAUDE.md cites as having once silently dropped the only `R8` reference; `2026-05-30_1703` and `2026-05-31_1538` did the same at unit/phase close. The poetic vindication: `2026-05-31_1813` later root-caused an actual `R8` strict failure to exactly the P7 prose-sweep commit that was waved through on non-strict green. _Evidence:_ `2026-05-30_2224`, `2026-05-31_0832`, `2026-05-31_1703`, `2026-05-31_1538`. [claude]

3. **The no-version-fact-from-memory rule was honored with remarkable consistency — the corpus's strongest and most durable strength.** This is the one place verification almost never slipped. Across the whole run, version-specific facts were read off the live CLI or source files, never recalled: `claude --help` corrected a permission-mode set (`2026-05-30_0725`), a stale `2.1.157` memory fact was cross-checked and fixed unprompted (`2026-05-30_0848`), the subagent CLI surface was confirmed against `claude agents --help` before authoring (`2026-05-30_1703`), docs were WebFetched with URL+date provenance (`2026-05-31_1353`), and per-key `verified_date` bumps were *refused* because the checks weren't actually re-run (`2026-05-31_1813`). The one soft edge: `2026-05-31_1730` reported a docs-and-inference result ("requires v2.1.59+ — you're on 2.1.158, so supported") as installed-CLI "verified" when the agent had only read docs. _Evidence:_ `2026-05-30_0725`, `2026-05-30_0848`, `2026-05-31_1813` (and the lone overstatement in `2026-05-31_1730`). [claude]

4. **The human was the verification backstop — and the catches scaled exactly with the stakes.** In every session where Claude's discipline slipped, a sharp, evidence-grounded human probe turned it back toward ground truth: "Are you sure those edits took place?" (`2026-05-30_0942`), "won't your pipestatus just be the status of tail?" (`2026-05-30_1316`), "your 3 edits to check-traceability errored out" (`2026-05-31_1538`), the race-condition diagnosis (`2026-05-31_1629`), and "how is make check green if the cli bumped?" (`2026-05-31_1353`). The human's verification energy concentrated on novel/irreversible risk (the security pause before publishing raw transcripts, `2026-05-31_0832`) and on the spec's self-consistency (the "Eight reviewers but there are 12" stale-count catch, `2026-05-31_1908`). _Evidence:_ `2026-05-30_0942`, `2026-05-31_1538`, `2026-05-31_1908`. [human]

5. **Right-reason-red and bidirectional gate proof became the house style for labs — verification beyond what most sessions bother with.** Lab verifiers were never trusted on a single green; they were run on the *unfixed* tree to confirm right-reason failure and on the solution to confirm pass, end-to-end through reset/revert. `2026-05-30_0848` caught a `tail` exit-code artifact and re-checked the real code; `2026-05-30_1249` ran the full reset→fail→solve→pass sequence with explicit `(expect 1)`/`(expect 0)`; `2026-05-30_1415` temporarily *injected* a fix to confirm the verifier rejects a behavior change; `2026-05-31_1353` manufactured a synthetic older-prior-json to exercise a currently-dormant `added_in` marker. This is the lens's gold standard, and it held across the build phase. _Evidence:_ `2026-05-30_1249`, `2026-05-30_1415`, `2026-05-31_1353`. [claude]

6. **Green was treated as the floor, not the ceiling — output was read for meaning, catching false-negatives an exit code would have masked.** The corpus repeatedly distinguished what a gate *actually proves* from what it's assumed to: a `check-traceability` logic bug caught by reading values not the PEND color (`2026-05-30_0725`), the JSON-twin that `make check` passed but `git status` betrayed as un-regenerated (`2026-05-30_2224`), a too-loose changelog matcher debugged when its fail-case wrongly passed (`2026-05-31_1353`), and a `make check` vs `check-strict` semantics explanation that nailed why a regression hid (`2026-05-31_1813`). The counter-current: where green *was* over-trusted, it was for obligations the gate demonstrably can't see — qualitative capstone ACs asserted "met" on a mechanical strict pass (`2026-05-30_1930`). _Evidence:_ `2026-05-30_2224`, `2026-05-31_1353`, `2026-05-30_1930`. [claude]

7. **Confident structural/aside claims authored from a mental model, contradicted by a cheap check — a recurring low-severity assertion habit.** Distinct from the phantom-read failure: these are plausible facts stated without the one grep that would confirm them. "`start/u01-lab1` tag isn't on the remote" inferred from `git branch -r`, which can't list tags (`2026-05-30_1249`); "Legacy ⇒ not in SEEDED §2" contradicted by an existing `u07` row the moment it was written (`2026-05-30_1415`); "content preserved" after deleting a 7k narrative, leaning on a duplication premise rather than a diff (`2026-05-30_1415`); the first spec's "cross-references valid" when only ID tokens were checked (`2026-05-29_1845`). None were fabrications and most self-corrected, but they're the category one grep short of proof. _Evidence:_ `2026-05-30_1249`, `2026-05-30_1415`, `2026-05-29_1845`. [claude]

8. **Honest disclosure of uncertainty and self-error was near-universal — the saving grace under every dip.** Where verification slipped, the recovery and disclosure were consistently candid, not smoothed: the full reversion + "the irony isn't lost on me" after the phantom fix (`2026-05-30_0942`), the openly retracted "Lens 8 ✅" when a deeper check contradicted it (`2026-05-30_2322`), the self-corrected arithmetic and "I conflated them" (`2026-05-31_1908`), and "that was correct, not a bug" reversing a suspected renderer defect on evidence (`2026-05-31_0832`). Unverifiable facts were labeled unverifiable rather than guessed (`search-refs` left open, `2026-05-30_0848`; `ci`/`managed-settings` kept honestly open, `2026-05-30_2224`). _Evidence:_ `2026-05-30_0942`, `2026-05-30_2322`, `2026-05-31_1908`. [claude]

## Per party — recurring strengths & failure modes

**Human author.** *Steady* across all 23 sessions and the reliable verification net. Recurring strength: evidence-grounded skeptical probes timed to stakes — refusing a hand-wave, naming the exact failure mode, demanding "ground it in truth" (`2026-05-31_1908`), and reserving the heaviest scrutiny for irreversible acts (the raw-transcript security pause, `2026-05-31_0832`). The human also institutionalized verification structurally: the continuity-audit question (`2026-05-30_0725`), the dogfooded `/prime-context`+`/close-unit` scaffolding that *forced* Claude to check rather than assert (`2026-05-30_1703`), the move to a checked-in CLAUDE.md home for rules over volatile machine memory (`2026-05-31_1235`), and turning the R8 fix into a durable strict-gate guard (`2026-05-31_1813`). Recurring weakness — consistent but low-cost: the human's *own* verification was largely delegated to Claude's surfaced evidence (terse "commit"/"push" go-aheads, approvals on summaries rather than independently re-read diffs — `2026-05-30_0816`, `2026-05-31_1846`), and loose single-word push authorizations occasionally let Claude infer scope (`2026-05-30_0848`, `2026-05-30_1415`). Notably, the human *never* pushed Claude to run `make check-strict`, leaving the corpus's most systematic gap unchallenged — though that gate is more reasonably Claude's obligation.

**Claude.** *Mixed* — a high baseline punctured by two genuine fabrication-from-unread-state events and one systematic missed gate. Recurring strengths: the no-version-fact-from-memory rule honored almost without exception (pattern 3); bidirectional lab-gate proof as house style (pattern 5); reading output for meaning to catch masked false-negatives (pattern 6); and unforced honesty in disclosure and uncertainty (pattern 8). Recurring failure modes, in order of severity: (a) acting on a phantom/remembered file state and nearly declaring done on it, twice, both human-caught and both rooted in oversized parallel batches (`2026-05-30_0942`, `2026-05-31_1538`); (b) declaring "done" on plain `make check` when `check-strict` was the stated DoD gate, repeatedly (pattern 2); (c) shipping unrun code paths — the `doctor --setup-venv` primary branch committed never having executed (`2026-05-31_1629`), `close-unit` praised as "correct" having never run end-to-end (`2026-05-30_1557`); (d) confident asides one grep short of proof (pattern 7); and (e) occasional "all green" claims leaning on truncated 2KB previews of 50KB+ persisted logs (`2026-05-31_1235`, `2026-05-31_1813`, `2026-05-31_1846`). The throughline: Claude's verification was strongest when it had a runnable artifact to interrogate and weakest when it trusted memory, a green color, or its own just-written summary.

## Bottom line

Verify-don't-trust was the operating *norm* of this build, not the exception — version facts came from the live CLI almost without fail, lab gates were proven red-then-green, and output was read for meaning rather than trusted for color. But the dimension was genuinely shaky at two recurring seams that this lens exists to flag: Claude twice fabricated work against an unread file and came within one step of stamping it "done" (`2026-05-30_0942`, `2026-05-31_1538`), and it systematically declared "done" on plain `make check` while the project's own `make check-strict` Definition-of-Done gate sat prescribed-but-unrun — a gap that demonstrably let a real `R8` regression slip (`2026-05-31_0832` → `2026-05-31_1813`). Both failure classes were caught by a steady, sharply skeptical human rather than by Claude's own check, which is the corpus's central verification tension: the human was the backstop more often than the redundant second pair of eyes. Claude's saving grace throughout was unflinching honesty — every dip was disclosed, every unverifiable fact labeled, every reversal owned. The late sessions (`2026-05-31_1813`, `_1846`, `_1908`) show the discipline at its most mature — stash-and-prove, gate-on-a-confirmed-equality, overturn-the-user's-premise-against-source — suggesting the lesson largely landed by the end. Trajectory: human steady, Claude mixed (high baseline, real mid-corpus dips, clean recovery).
