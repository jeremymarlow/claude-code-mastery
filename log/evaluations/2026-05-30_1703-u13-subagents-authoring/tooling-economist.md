## What worked / what didn't (both parties)

**Claude — strengths.** Tool selection was the headline: skill, command, and AskUserQuestion each appeared exactly where it earned its keep, and never gratuitously. The live-CLI verification habit (subagent surface, hook schema, MCP `✓ Connected`, headless flags) is the economical route to correctness and was applied with real rigor — the MCP server was *proven* against the installed CLI, not asserted. Bash probes were well-batched. The session shipped four full units, two real fixtures (`tools/check-on-edit`, `taskflow_mcp.py`), and a wired in-session hook that closed a long-standing open loop (L2), all with `make check` green at every gate, and the running commentary was clear and accurately scoped.

**Claude — weaknesses.** The hand-maintained `meta/version-data.json` twin is dead weight no tool reads, yet it doubled every vd edit and produced one read-before-write rework turn; Claude served the duplication faithfully instead of flagging that it should be generated. Opus ran a great deal of deterministic state-sync chore that a cheaper model or tighter script could have carried. And the `update-config` skill, while correct, was the heaviest call of the session for a thin slice of extracted facts — fine here on budget, but worth being deliberate about.

**Human — strengths.** Per-change commit/push approvals, never standing — clean adherence to the project's own working agreement. The closing branch-nomenclature exchange showed genuine engineering judgment: the author overrode Claude's recommendation with a better-reasoned call and Claude concurred. Direction was terse and unambiguous ("go ahead with U13", "commit and push, then start U15").

**Human — weaknesses.** The argument-less `/close-unit` invocation created avoidable redundancy directly after Claude had already done the work; it cost a verification pass. Minor, and well-absorbed.

## Bottom line

A model session for the Tooling Economist's lens: the right capability was reached for at each decision point — skill for the authoritative hooks schema, command for the chore, AskUserQuestion for the one real fork — and version-specifics were proven against the live CLI rather than recalled, including an MCP server verified to `✓ Connected`. The economy faults are narrow and real: a hand-maintained JSON twin that no tool consumes (and that cost a rework turn), and a strong model spent on a large amount of deterministic state-sync that scripting or a cheaper model would have carried. Neither dented the output — four units and Phase 5 landed green and honest, with debt left openly deferred rather than faked. The human ran a disciplined, well-gated loop and out-reasoned Claude on the one branch-naming call that mattered.
