## What worked / what didn't (both parties)

**Claude — worked.** This is a textbook execution of a bounded build slice under an approved spec. It oriented from the canonical first-read surfaces before touching anything, traced the unit against design/catalog/coverage, sliced the lab into a clean start-state commit and a separate solution branch, and — most impressively — *proved* the verifier gates by running reset → fail-clean → pass-on-solution end-to-end. State hygiene was the standout: every parallel surface (§3, tasks.md, decisions.md rows, the L7 ledger, SEEDED.md, project memory) moved together, with the no-new-debt reasoning made explicit. Nothing was built ahead of the plan; nothing was bundled into an unreviewable lump.

**Claude — didn't.** Two soft spots, both process-discipline rather than correctness. (1) It gated "done" on `make check` and never ran `make check-strict`, the documented Definition-of-Done gate — defensible for a P5 increment but unflagged, so a reviewer can't tell whether the skip was reasoned or forgotten. (2) On a scoped "commit and push," it published two extra refs (solution branch + tag); justified and transparently flagged *after* the fact, but a stricter reading of the per-change-approval rule would have asked first. Neither caused harm; both are the difference between very-good and airtight.

**Human — worked.** The human ran this exactly the way a spec-driven repo is meant to be run: a terse "continue with the next unit" trusting Claude to read §3 and pick up U5, then a firm hold on the push gate (no standing permission inferred), then scoped, discrete go-aheads. They also actively maintained state quality — catching the venv-note wording so the persistent memory stays honest for the next session. The interrupt-and-correct on the venv was efficient.

**Human — didn't.** Little to fault through this lens. The one thing the human *could* have done is forced the `check-strict` question — but that's Claude's documented obligation to surface, not the human's to police. The "commit and push" instruction was slightly under-specified about which refs, which is what gave Claude room to interpret; naming "and the lab refs" would have removed all ambiguity.
