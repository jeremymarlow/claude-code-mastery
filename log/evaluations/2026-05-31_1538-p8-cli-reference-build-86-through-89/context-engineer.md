## What worked / what didn't (both parties)

**The human author** ran this session the way a context engineer would want: a real priming command that establishes ground truth from files before any action, tight per-slice scoping so each unit of work fit cleanly in the window, and surgical interventions that repeatedly yanked Claude back to the source of truth at exactly the right moments. The push gate was honored to the letter. There is essentially nothing to fault on the human side through this lens — the author even nudged Claude toward the correct tool (Read over `awk`) when the model was flailing.

**Claude** did the substantive work correctly — the 8.6 dogfood pointers, the 8.7 dynamic-discovery refactor, the 8.8 playbook, and the 8.9 state sync are all well-scoped, traceable, and `make check`-green, and the close-out externalized state properly. Its priming and its candor were genuine strengths. But the session's context management was poor in two linked ways: (a) it repeatedly fired oversized parallel batches into a flaky console, polluting the window with dozens of duplicate/wasted reads, and (b) — far worse — when that corrupted its picture of a file, it edited from the corrupted recollection instead of re-Reading, exactly the failure the project memory exists to prevent. The recovery discipline was sound, but recovery is not prevention, and the same root cause recurred at least three times after Claude had explicitly diagnosed it. A genuinely context-disciplined session would have, at the first "Wasted call," dropped to single scoped calls and re-Read before any edit — and would never have authored an edit against unseen file contents.

## Bottom line

A session whose *content* was correct and whose *orientation and state-externalization* were exemplary, undermined by a persistent self-inflicted batching problem that flooded the window with redundant reads, triggered an API cascade, and — most damningly — led Claude to edit a file from a fabricated mental model that the human had to catch. The through-line: priming and close-out hygiene were textbook, but the middle violated the repo's own first commandment — trust the files, not memory — and Claude knew it was doing so before it stopped. The human's tight scoping and well-aimed interventions are what kept the session on the rails.
